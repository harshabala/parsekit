## Tone and voice

### Words to use

Sparse, precise, active voice. One idea per line. Numbers over adjectives. Understated confidence — the tool works quietly in the background.

### Words to avoid

"Simply", "just", "powerful", "seamless", "luxury", "exclusive", and any cloud/AI hype. No exclamation marks in UI copy.

## Usage dos and don'ts

**Do:**

- Use CSS variables from `src/index.css` for all colors.
- Respect `prefers-color-scheme` — the app follows system light/dark.
- Use `--font-serif` only for the app title and about header.
- Use `font-mono tabular-nums` for file counts and numeric controls.

**Don't:**

- Reintroduce system blue (`#007aff`) accents.
- Hardcode hex in Svelte components.
- Use `transition: all` — specify properties explicitly.
